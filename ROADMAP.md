# Roadmap

## Current Status

Nargis has a working multi-service foundation with the core product loops in place and an evolving AI/runtime layer. The current focus is maintainability, clearer service boundaries, and improving the depth of the assistant experience without destabilizing the core workflows.

## Phase 1: Foundation

Status: complete

- Monorepo and shared tooling established
- Web, Python, and Go services integrated
- Core productivity flows implemented
- Realtime voice-to-response path working end to end
- Baseline testing and CI flows established

## Phase 2: Agent and Platform Maturity

Status: in progress

- Improve service boundaries and shared runtime contracts
- Strengthen persistence, auth, and operational configuration
- Continue moving backend logic into reusable service modules
- Expand typed contracts and reduce duplicated integration logic
- Keep the local developer workflow fast while tightening quality gates

## Phase 3: Proactive Assistance

Status: planned

- Smarter task capture and execution flows
- Better memory and context handling for assistant interactions
- Productivity coaching, summaries, and insight generation
- More proactive scheduled and event-driven assistance

## Near-Term Priorities

- Continue hardening deployment and environment conventions
- Improve backend persistence depth and long-term data handling
- Tighten documentation so repo entrypoints and operational docs stay aligned
- Keep reducing oversized modules and duplicated runtime logic

## Backlog Themes

- Richer AI-assisted prioritization and planning
- Stronger journal intelligence and semantic retrieval
- Better observability and production monitoring
- More polished cross-device and responsive UX
